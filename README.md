## 搬瓦工 CN2 是什么，先弄清楚概念

CN2 是中国电信旗下的高端国际线路，全称 China Telecom Next Generation Carrier Network。

搬瓦工目前在售的套餐里有两类 CN2：

- **CN2 GT（Global Transit）**：国际段走 CN2，但国内回程退回普通 163 骨干网，晚高峰拥堵问题和普通线路差距不大，属于半程优化
- **CN2 GIA（Global Internet Access）**：去程和回程全程走 CN2 GIA 网络，三网回程都经 AS4809，延迟低、丢包少，是目前能买到的对国内最友好的国际线路之一

很多人搜"搬瓦工 CN2 不通"，其实买的是 CN2 GT 套餐（最便宜的那档，$49.99/年），这个套餐本身没什么问题，但晚上 8 点到 11 点偶尔会有拥堵感。

真正的 CN2 GIA-E（电商套餐）是独立的一系列套餐，季付 $49.99 起，可切换 12 个以上机房，这两个不要搞混。

## 第一步：先判断是哪种"不通"

搬瓦工 CN2 不通，大致分三类情况，排查方向完全不同：

### 情况一：国内国外全 ping 不通

你的 IP 在国内 ping 不通，用境外服务器测也 ping 不通，说明 **VPS 本身有问题**，和 CN2 线路无关。

最常见原因：VPS 没有正常启动。进 KiwiVM 控制面板，看一眼 VPS 的运行状态就清楚了。

**解决步骤：**

1. 登录 KiwiVM 控制面板（地址在购买邮件里）
2. 主界面顶部显示 VPS 当前状态，如果是 `stopped` 或者异常，点 `Start` 或 `Restart`
3. 等大约 30 秒，再 ping 一次你的 IP

大部分情况这一步就解决了。不行的话进 `Emergency Console` 看一下系统里有没有报错。

还有一种情况是 iptables 配置写错了，把所有入站流量都 drop 掉了——这时 VPS 本身在跑，但任何连接都进不来。进 VNC/Emergency Console 执行 `iptables -F` 清掉规则，确认正常后再重新配防火墙。

### 情况二：国外能 ping 通，国内 ping 不通

这个几乎可以直接判断：**IP 被国内屏蔽了**。

IP 被封在搬瓦工这个价位的 VPS 里算常见事，特别是 IP 之前的使用记录不干净的情况下。搬瓦工对被封 IP 提供两个处理方式：

- **免费换 IP**：KiwiVM 后台 → `IP Replacement` → 每隔一定周期可以免费换一个同机房 IP。如果新 IP 国内通了，问题解决。
- **换机房**：KiwiVM 后台 → `Migrate to another DC` → 选一个其他机房迁过去。CN2 GIA-E 套餐支持十几个机房切换，换一个之后 IP 自动变，基本能解决。

换机房这个操作免费，数据全保留，迁移时间从几分钟到一个小时不等，迁完会收到邮件通知。

### 情况三：能连上但速度很慢、丢包高、晚上尤其明显

这不是"不通"，而是线路拥堵或者受到攻击导致的质量问题。

首先去 **bwhstatus.com** 看一眼。这是搬瓦工的官方状态页，如果当前有机房或线路故障，会在这里标注。有公告的话等一下就行，这是搬瓦工在处理的事。

没有公告的话，可能是你的 IP 遇到 DDoS 攻击被 nullroute 了，也就是流量被黑洞路由扔掉了。这种情况 IP 通常会暂时不通，等攻击结束会自动恢复，一般 24 小时内。

如果短时间内反复出现，建议换一个机房试试，DC6 和 DC9 之间可以互切，也可以切到日本软银 JPOS_1 或者荷兰 EUNL_9，这些机房的抗干扰能力各有侧重。

## 第二步：用 traceroute 验证线路是否真的走 CN2

很多人以为买了 CN2 套餐就一定走 CN2，其实要验证一下才放心。

在 VPS 上跑下面这段（要先安装 NextTrace）：

bash
# 安装 NextTrace
curl nxtrace.org/nt | bash

# 测试到电信的回程路由
nexttrace 219.141.136.10

# 测试到联通的回程路由
nexttrace 202.106.0.20

# 测试到移动的回程路由
nexttrace 221.179.155.161


看输出里的 IP 段：

| 看到什么 | 走的是什么线路 |
|---|---|
| 全程 `59.43.x.x` / AS4809 | CN2 GIA，最好 |
| 前段 `59.43.x.x`，后段 `202.97.x.x` | CN2 GT，半程 |
| 全程 `202.97.x.x` | 普通 163 骨干，最差 |

如果买的是 CN2 GIA-E 套餐，跑出来全程都是 AS4809，才算正常。如果出来的是 202.97 开头，要么套餐买错了，要么机房迁移有问题，联系客服确认一下。

## 买错套餐是很多人忽略的问题

讲真，搬瓦工不通这个问题有相当一部分是买错了套餐。

最常见的坑是把 CN2 GT 套餐当 CN2 GIA 买。两者价格差了一个档次——GT 最便宜是 $49.99/年，GIA-E 是 $49.99/季，差了 4 倍。如果你是在某个地方看到"搬瓦工 CN2 套餐年付 49 刀"然后冲进去买的，大概率买到的是 GT，不是 GIA。

GT 不是不能用，日常速度可以，就是高峰期体验和 GIA-E 明显有差距。如果你当时选择的机房显示的是 USCA_3（DC3 CN2）或者 DC8，那就是 CN2 GT。

另外要注意：部分旧的 CN2 套餐迁移到 CN2 机房之后，**流量会按 0.33x 倍率计算**——也就是原本 1TB 流量迁过去之后只剩 330GB 可用。CN2 GIA-E 套餐不受这个限制，迁哪里都是满流量。

## FAQ

**Q：我的搬瓦工 VPS 在国内完全 ping 不通，但 KiwiVM 后台显示运行正常，是什么原因？**

A：八成是 IP 被墙了。进 KiwiVM 后台的 `IP Replacement` 申请换 IP，或者用 `Migrate to another DC` 换机房，IP 会跟着换，国内重新测试通了就是这个原因。

**Q：买了 CN2 GIA-E 套餐，晚上速度还是慢，正常吗？**

A：CN2 GIA-E 在晚高峰比 CN2 GT 表现稳很多，但偶发的拥堵是正常的，毕竟中美之间的海底光缆容量有限。先去 bwhstatus.com 确认没有机房公告，再用 traceroute 验证回程路由没有跑偏，两步确认下来基本可以定位是否是线路问题。DC6 和 DC9 互切也是个常用操作。

**Q：CN2 GIA-E 套餐的 12 个机房，每个机房的线路都一样吗？**

A：不一样。DC6（USCA_6）和 DC9（USCA_9）是电信/联通 CN2 GIA + 移动 CMIN2；日本软银 JPOS_1 是三网软银线路；荷兰 EUNL_9 是联通 AS9929。一个套餐包含这么多机房选择，根据自己的运营商迁到最合适的那个就行。

**Q：搬瓦工的退款怎么操作？**

A：在 30 天内，进 KiwiVM 客服工单提交退款请求即可，说明 `refund request` 并附上订单号。官方说明是全额退款，我见过的案例都是处理在 5 个工作日内完成。

**Q：搬瓦工官网我访问不了怎么办？**
